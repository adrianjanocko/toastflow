# Headless Slot

Use the `ToastContainer` slot in Vue or Nuxt when you like Toastflow's runtime behavior but need your own card markup.

## Basic Slot

```html [App.vue]
<ToastContainer v-slot="{ toast, ui }">
  <div v-bind="ui.wrapperProps">
    <article v-bind="ui.getRootProps({ class: 'my-toast' })">
      <strong>{{ toast.title }}</strong>
      <p v-if="toast.description">{{ toast.description }}</p>

      <button v-bind="ui.getCloseProps()">Close</button>
    </article>
  </div>
</ToastContainer>
```

## Actions And Progress

::code-group

```html [actions.vue]
<button
  v-for="button in ui.buttons.items"
  :key="button.id"
  v-bind="ui.getButtonProps(button)"
>
  {{ button.label }}
</button>
```

```html [progress.vue]
<div v-if="ui.progress.show" v-bind="ui.progress.getTrackProps()">
  <div v-bind="ui.progress.getBarProps()" />
</div>
```

```html [close.vue]
<button v-bind="ui.getCloseProps({ class: 'close-button' })">Close</button>
```

::

## What `ui` Gives You

::card-group
::card{title="Accessibility props" icon="i-tabler-shield-check"}
`wrapperProps` and `getRootProps()` keep structure, roles, labels, and live-region behavior intact.
::

::card{title="Dismiss behavior" icon="i-tabler-x"}
`getCloseProps()` wires the button to the correct toast id.
::

::card{title="Action buttons" icon="i-tabler-hand-click"}
`getButtonProps(button)` preserves click handling, dismiss-on-click, and ARIA labels.
::

::card{title="Progress helpers" icon="i-tabler-gauge"}
`progress.*` helpers let you keep Toastflow's timer state while drawing your own bar.
::
::

## Slot Props Reference

The `ToastContainer` default slot provides these props:

| Prop                     | Type            | Use                                        |
| ------------------------ | --------------- | ------------------------------------------ |
| `toast`                  | `ToastInstance` | Current toast data                         |
| `ui`                     | `ToastUI`       | Structured helpers (preferred API)         |
| `progressResetKey`       | `number`        | Changes when timer/progress should restart |
| `duplicateKey`           | `number`        | Changes on duplicate event                 |
| `updateKey`              | `number`        | Changes on update event                    |
| `bumpAnimationClass`     | `string`        | Duplicate bump animation class             |
| `clearAllAnimationClass` | `string`        | Clear-all animation class                  |
| `updateAnimationClass`   | `string`        | Update animation class                     |
| `dismiss`                | `(id) => void`  | Dismiss helper                             |

### `ui` Highlights

- **`wrapperProps`** / **`getRootProps()`** — Structure, roles, labels, live-region
- **`getCloseProps()`** — Close button wiring
- **`getButtonProps(button)`** — Action button click + ARIA
- **`progress.show`**, **`progress.trackProps`**, **`progress.barProps`** — Timer-aware progress helpers
- **`buttons.items`** — Resolved button array
- **`icon`**, **`createdAt`** — Derived rendering state

## Slot vs Core

::accordion
::accordion-item{label="Use the slot"}
Use it when you are in Vue or Nuxt and only want to replace markup/styles.
::

::accordion-item{label="Use toastflow-core"}
Use it when you are outside Vue or building an entirely custom renderer.
::
::
