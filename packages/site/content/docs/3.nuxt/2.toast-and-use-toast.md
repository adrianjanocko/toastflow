Both are auto-imported by `nuxt-toastflow`. They point to the same Toastflow runtime.

::card-group
::card{title="Use toast" icon="i-tabler-bolt"}
Best for one-off calls in pages, click handlers, composables, and callbacks.
::

::card{title="Use useToast()" icon="i-tabler-braces"}
Best when you want to name the API once and pass it around explicitly.
::
::

## Side By Side

::code-group

```ts [toast.ts]
toast.success({
  title: "Saved",
});
```

```ts [use-toast.ts]
const toastApi = useToast();

toastApi.success({
  title: "Saved",
});
```

::

## Same Store

```ts [same-store.ts]
const toastApi = useToast();

toastApi.dismissAll();
toast.dismissAll();
```

Both calls operate on the same queue, timers, subscriptions, and visible toasts.

## Common Nuxt Patterns

::tabs
::tabs-item{label="Click handler"}

```vue [components/CopyLink.vue]
<script setup lang="ts">
function copyLink() {
  toast.success({ title: "Copied" });
}
</script>

<template>
  <UButton @click="copyLink">Copy link</UButton>
</template>
```

::

::tabs-item{label="Composable"}

```ts [composables/useSaveProfile.ts]
export function useSaveProfile() {
  const toastApi = useToast();

  return async function saveProfile(payload: unknown) {
    await toastApi.loading(
      $fetch("/api/profile", { method: "POST", body: payload }),
      {
        loading: { title: "Saving profile" },
        success: { title: "Profile saved" },
        error: { title: "Save failed" },
      },
    );
  };
}
```

::

::tabs-item{label="Update by id"}

```ts [update-by-id.ts]
const id = toast.show({
  type: "loading",
  title: "Uploading",
  duration: 0,
});

toast.update(id, {
  type: "success",
  title: "Upload complete",
  duration: 3000,
});
```

::
::

## Client-Side Rule

::warning{icon="i-tabler-server-off"}
Do not call `toast` or `useToast()` from Nitro routes, server handlers, route middleware that only runs on the server, or SSR-only code. Toasts are UI feedback, so trigger them from client execution.
::

## Quick Decision

::accordion
::accordion-item{label="I am inside a Vue component event handler"}
Use `toast`. It keeps the file short and easy to scan.
::

::accordion-item{label="I am writing a reusable composable"}
Use `useToast()` when a named local API makes the composable clearer.
::

::accordion-item{label="I need a lower-level import"}
Use `nuxt-toastflow/runtime` for explicit runtime imports. See [Runtime exports](/docs/api/runtime-exports).
::
::
