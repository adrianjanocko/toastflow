`nuxt-toastflow` adds Nuxt-specific wiring around the shared Toastflow config.

::field-group

::field{name="config" type="ToastConfig"}
Shared Toastflow runtime config. Default: `{}`.
::

::field{name="css" type="boolean"}
Toggle shipped CSS injection. Default: `true`.
::

::field{name="componentName" type="string | false"}
Rename or disable the auto-registered container. Default: `"ToastContainer"`.
::

::field{name="components" type="boolean"}
Auto-register the remaining components (`Toast`, icons, …). Default: `true`.
::

::

## Full Example

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["nuxt-toastflow"],
  toastflow: {
    css: true,
    componentName: "ToastContainer",
    config: {
      position: "top-right",
      alignment: "left",
      duration: 5000,
      maxVisible: 5,
      queue: true,
      pauseOnHover: true,
    },
  },
});
```

## Option Details

::accordion
::accordion-item{label="config"}
Everything you would pass to `createToastflow(...)` in Vue goes under `toastflow.config`.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["nuxt-toastflow"],
  toastflow: {
    config: {
      position: "bottom-right",
      duration: 4000,
      maxVisible: 3,
      queue: true,
    },
  },
});
```

Keep this JSON-serializable. Runtime callbacks belong on individual toast calls.
::

::accordion-item{label="css"}
Leave `css: true` for the normal path. Set it to `false` when you want to own stylesheet loading.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["nuxt-toastflow"],
  css: ["~/assets/toastflow.css"],
  toastflow: {
    css: false,
  },
});
```

::

::accordion-item{label="componentName"}
Rename the auto-registered component:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["nuxt-toastflow"],
  toastflow: {
    componentName: "AppToasts",
  },
});
```

Or disable registration and import explicitly:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["nuxt-toastflow"],
  toastflow: {
    componentName: false,
  },
});
```

The registered component supports the same `id` prop as in Vue, so you can mount several containers and route toasts with `containerId` — see [Multiple Containers](/docs/vue/multiple-containers).
::

::accordion-item{label="components"}
By default the module also auto-registers `Toast`, `ToastProgress`, and the icon components (`ArrowPath`, `Bell`, `CheckCircle`, `InfoCircle`, `QuestionMarkCircle`, `XCircle`, `XMark`) under their export names — useful for headless slots and custom rendering.

The icon names are generic; if one collides with your own components or an icon library, turn the registration off and import what you need from `nuxt-toastflow/runtime`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["nuxt-toastflow"],
  toastflow: {
    components: false,
  },
});
```

```vue [app/components/MyToast.vue]
<script setup lang="ts">
import { CheckCircle } from "nuxt-toastflow/runtime";
</script>
```

::

## JSON-Serializable Config

::warning
Do not put functions in `toastflow.config`. Nuxt config is serialized. Use per-toast callbacks like `onClose`, `onClick`, or button `onClick` at runtime.
::

::code-tree{default-value="runtime/toast.ts" expand-all}

```ts [runtime/toast.ts]
toast.success({
  title: "Saved",
  onClose() {
    console.log("toast closed");
  },
});
```

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  toastflow: {
    config: {
      onClose() {
        console.log("not serializable here");
      },
    },
  },
});
```
