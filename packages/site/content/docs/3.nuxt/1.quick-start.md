Use `nuxt-toastflow` for Nuxt 3 and Nuxt 4 apps. The module wires the plugin, CSS, auto-imports, component registration, and the same runtime exports available from `vue-toastflow`.

::steps

### Install

::code-group

```bash [pnpm]
pnpm add nuxt-toastflow
```

```bash [npm]
npm install nuxt-toastflow
```

```bash [yarn]
yarn add nuxt-toastflow
```

::

### Enable the module

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["nuxt-toastflow"],
  toastflow: {
    config: {
      position: "top-right",
      duration: 5000,
      maxVisible: 5,
    },
  },
});
```

### Render the container once

```vue [app.vue]
<template>
  <ToastContainer />
  <NuxtPage />
</template>
```

One container is the default setup. You can also mount additional `<ToastContainer id="...">` instances and route toasts to them with the `containerId` option — see [Multiple Containers](/docs/vue/multiple-containers).

### Use `toast` or `useToast()`

```vue [pages/index.vue]
<script setup lang="ts">
toast.success({
  title: "Saved",
  description: "Your changes are live.",
});

const toastApi = useToast();

toastApi.info({
  title: "useToast() works too",
});
</script>
```

::

::warning
Use `toast` and `useToast()` from client-side execution: click handlers, `onMounted`, client composables, and UI callbacks. Do not call them from Nitro handlers.
::

## Minimal File Tree

::code-tree{default-value="nuxt.config.ts" expand-all}

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["nuxt-toastflow"],
  toastflow: {
    config: {
      position: "top-right",
      queue: true,
    },
  },
});
```

```vue [app.vue]
<template>
  <ToastContainer />
  <NuxtPage />
</template>
```

```vue [pages/index.vue]
<script setup lang="ts">
async function save() {
  await toast.loading($fetch("/api/save"), {
    loading: { title: "Saving" },
    success: { title: "Saved" },
    error: { title: "Failed" },
  });
}
</script>

<template>
  <UButton @click="save">Save</UButton>
</template>
```

::

## What The Module Gives You

::card-group
::card{title="Auto-imported toast" icon="i-tabler-bell"}
Call `toast.success(...)`, `toast.loading(...)`, or `toast.update(...)` without a manual import.
::

::card{title="Auto-imported useToast()" icon="i-tabler-braces"}
Grab the same API once inside a component or composable and reuse it locally.
::

::card{title="Auto-registered ToastContainer" icon="i-tabler-layout-navbar"}
Render `<ToastContainer />` in `app.vue` without importing it.
::

::card{title="Vue runtime exports" icon="i-tabler-package" to="/docs/api/runtime-exports"}
Use `nuxt-toastflow/runtime` when you need explicit imports for `Toast`, `ToastProgress`, types, or lower-level runtime pieces.
::
