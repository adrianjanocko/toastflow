Toastflow resolves config in three layers:

::steps

### Internal defaults

Shipped by `toastflow-core`.

### Global config

Passed to `createToastflow(...)` (Vue) or `createToastStore(...)` (headless).

### Per-toast overrides

Passed to `show`, typed helpers, `loading`, or `update`.
::

::note
**Full TypeScript definitions:** [`packages/core/src/types.ts`](https://github.com/adrianjanocko/toastflow/blob/main/packages/core/src/types.ts){target="\_blank"}. The reference below covers the most common options.
::

## Layout & Position

::field-group

::field{name="position" type="ToastPosition"}
Viewport anchor for the stack. Default: `"top-right"`.
::

::field{name="alignment" type='"left" | "center" | "right"'}
Text direction inside the toast body. Default: `"left"`.
::

::field{name="offset" type="string"}
Distance from the viewport edge. Default: `"16px"`.
::

::field{name="gap" type="string"}
Gap between toasts. Default: `"8px"`.
::

::field{name="width" type="string"}
Default toast width. Default: `"350px"`.
::

::field{name="zIndex" type="number"}
Root container z-index. Default: `9999`.
::

::field{name="overflowScroll" type="boolean"}
Scroll the stack when it grows taller than the viewport. Default: `false`.
::

::field{name="containerId" type="string"}
Target a specific `<ToastContainer id>` ([details](/docs/vue/multiple-containers)). No default.
::

::

## Timing & Progress

::field-group

::field{name="duration" type="number"}
Auto-dismiss delay in ms. Default: `5000`. `Infinity` / `0` = no auto-dismiss.
::

::field{name="pauseOnHover" type="boolean"}
Pause the timer on hover or touch. Default: `true`.
::

::field{name="pauseStrategy" type='"resume" | "reset"'}
`"resume"` continues the remaining time, `"reset"` restarts the full duration. Default: `"resume"`.
::

::field{name="progressBar" type="boolean"}
Show a progress bar for finite durations. Default: `true`.
::

::field{name="progressAlignment" type='"right-to-left" | "left-to-right"'}
Progress animation direction. Default: `"right-to-left"`.
::

::field{name="showCreatedAt" type="boolean"}
Show a timestamp badge on the toast. Default: `false`.
::

::field{name="createdAtFormatter" type="(createdAt: number) => string"}
Function formatting the timestamp ([see below](#timestamps)). Default: locale time.
::

::

## Stacking & Queue

::field-group

::field{name="maxVisible" type="number"}
Max visible toasts per position. Default: `5`. `<= 0` = no cap.
::

::field{name="queue" type="boolean"}
Queue overflowed toasts instead of evicting. Default: `false`.
::

::field{name="order" type='"newest" | "oldest"'}
Insert order and eviction target. Default: `"newest"`.
::

::field{name="preventDuplicates" type="boolean"}
De-duplicate by container + position + type + title + description. Default: `false`.
::

::

## Interaction

::field-group

::field{name="closeButton" type="boolean"}
Show the floating close button. Default: `true`.
::

::field{name="closeOnClick" type="boolean"}
Dismiss on body click. Default: `false`.
::

::field{name="swipeToDismiss" type="boolean"}
Dismiss with a right swipe/drag. Default: `false`.
::

::

## Content & Appearance

::field-group

::field{name="showIcon" type="boolean"}
Show the type icon. Default: `true`.
::

::field{name="supportHtml" type="boolean"}
Allow HTML in title and description — use trusted content only. Default: `false`.
::

::field{name="sanitizer" type="(html: string) => string"}
Hook sanitizing HTML before render, e.g. DOMPurify ([details](/docs/global/styling#sanitizer-hook)). No default.
::

::field{name="css" type="ToastCSSOverrides"}
CSS variable overrides (`bg`, `color`, `accentColor`, …). Works globally and per-toast — per-toast values merge over the global ones ([details](/docs/global/styling)). No default.
::

::

## Per-Toast Only Options

Available on `show`, typed helpers, and `update` — not in global config:

::field-group

::field{name="theme" type="string"}
CSS accent class, e.g. `"brand"` → `tf-toast-accent--brand` ([details](/docs/global/styling#the-three-levels)).
::

::

## Example

::code-group

```ts [global-config.ts]
import { createToastflow } from "vue-toastflow";

app.use(
  createToastflow({
    position: "top-right",
    duration: 5000,
    maxVisible: 4,
    queue: true,
    pauseStrategy: "resume",
    preventDuplicates: true,
  }),
);
```

```ts [per-toast-override.ts]
toast.warning({
  title: "Server maintenance",
  position: "bottom-center",
  duration: 15000,
  closeOnClick: true,
  buttons: {
    alignment: "bottom-right",
    buttons: [{ label: "Dismiss" }],
  },
});
```

::

## Animation Class Names

Override globally or per-toast:

```ts [animation-config.ts]
createToastflow({
  animation: {
    name: "MyToastMotion",
    bump: "MyToastBump",
    clearAll: "MyToastClearAll",
    update: "MyToastUpdate",
  },
});
```

Defaults: `Toastflow__animation`, `Toastflow__animation-bump`, `Toastflow__animation-clearAll`, `Toastflow__animation-update`.

## Timestamps

`showCreatedAt: true` renders a small badge with the time the toast was created. The default format is the locale time; pass `createdAtFormatter` to change it:

```ts [created-at-formatter.ts]
createToastflow({
  showCreatedAt: true,
  createdAtFormatter: (createdAt) =>
    new Date(createdAt).toLocaleTimeString([], {
      hour: "2-digit",
      minute: "2-digit",
    }),
});
```

::warning
`createdAtFormatter` is a function, so in Nuxt it cannot live in `nuxt.config` (config is JSON-serialized). Configure it at runtime instead — see [Nuxt notes](/docs/nuxt/nuxt-notes).
::

## Buttons Config

```ts [buttons-config.ts]
toast.info({
  title: "Changes saved",
  buttons: {
    alignment: "bottom-right",
    layout: "row",
    buttons: [
      {
        label: "Undo",
        onClick(ctx) {
          console.log("Undo", ctx.id);
        },
      },
    ],
  },
});
```

Each button supports `label` or `html`, plus optional `id`, `ariaLabel`, `className`, `dismissOnClick` (dismiss before `onClick` runs), `dismissAfterClick` (dismiss after `onClick` runs), and `onClick(ctx, event)`.

::caution
`html` buttons are rendered without sanitization — pass trusted markup only, or sanitize it yourself. Set `ariaLabel` on `html` buttons so screen readers can announce them.
::
