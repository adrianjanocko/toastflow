Actions are the mutating methods on `toast` (Vue/Nuxt) or `store` (headless).

::note
**Full method signatures:** [`packages/core/src/store.ts`](https://github.com/adrianjanocko/toastflow/blob/main/packages/core/src/store.ts){target="\_blank"}.
::

## Method Overview

::field-group

::field{name="show" type="(options | title, options?) => ToastId"}
Create a toast, returns `ToastId`.
::

::field{name="loading" type="(input, config) => Promise"}
Bind an async flow to loading → success / error ([see below](#loading-helper)).
::

::field{name="update" type="(id, options) => void"}
Patch an existing toast in place.
::

::field{name="dismiss" type="(id) => void"}
Dismiss one toast.
::

::field{name="dismissAll" type="(filter?: { containerId?: string }) => void"}
Dismiss all visible and queued toasts, optionally scoped to a container.
::

::field{name="pause" type="(id) => void"}
Pause the timer for one toast.
::

::field{name="resume" type="(id) => void"}
Resume the timer using `pauseStrategy`.
::

::field{name="pauseQueue" type="() => void"}
Pause queue processing globally.
::

::field{name="resumeQueue" type="() => void"}
Resume queue processing.
::

::field{name="getState" type="() => ToastState"}
Current state snapshot.
::

::field{name="getConfig" type="() => ToastConfig"}
Resolved global config.
::

::field{name="subscribe" type="(listener) => () => void"}
Continuous state updates, returns an unsubscribe function.
::

::field{name="subscribeEvents" type="(listener) => () => void"}
One-off event stream ([details](/docs/api/events)).
::

::

Typed wrappers: `default`, `success`, `error`, `info`, `warning`, `custom` — they pin the `type` field automatically.

## Create Call Styles

| Style                      | Use when                                     |
| -------------------------- | -------------------------------------------- |
| `toast.show({ ... })`      | You already have a full payload object       |
| `toast.show(title, opts)`  | The title is the primary thing being shown   |
| `toast.success({ ... })`   | The toast type is known at the call site     |
| `toast.error(title, opts)` | You want a typed helper with title + options |

See [Toasts](/docs/vue/toasts#create-toasts) for runnable examples.

::warning
`title` or `description` must be non-empty. Empty content throws.
::

## Loading Helper

```ts [loading.ts]
await toast.loading(new Promise((resolve) => window.setTimeout(resolve, 900)), {
  loading: { title: "Saving" },
  success: { title: "Saved" },
  error: { title: "Failed" },
});
```

The loading toast uses `duration: Infinity` and `progressBar: false`. On resolve/reject, the same toast id transitions to success or error.

For real requests, pass `fetch`, `$fetch`, or a function that returns a promise. `success` and `error` can also be functions when the final message needs response data.

::warning
`loading(...)` returns the wrapped promise — it **rejects** when the task fails (the error toast is shown either way). If you fire-and-forget the call, attach a `.catch(() => {})` or the rejection surfaces as an unhandled promise rejection.
::

## Update

```ts [update.ts]
const id = toast.show({
  type: "loading",
  title: "Uploading",
  description: "Waiting for the update call...",
  duration: 0,
  progressBar: false,
});

window.setTimeout(() => {
  toast.update(id, {
    type: "success",
    title: "Upload complete",
    description: "The same toast id was patched in place.",
    duration: 3500,
    progressBar: true,
  });
}, 900);
```

<DocsToastPreview variant="update" />

- Visible toast timer is re-scheduled on update.
- Queued toast instance is replaced in queue.

## Dismiss

::code-tree{default-value="dismiss-one.ts" expand-all}

```ts [dismiss-one.ts]
toast.dismiss(id);
```

```ts [dismiss-all.ts]
toast.dismissAll();
```

```ts [dismiss-container.ts]
toast.dismissAll({ containerId: "panel" }); // only container "panel"
```

::

`dismiss` fires `onClose`, marks the toast as `leaving`, then removes it after the leave delay. `dismissAll` marks visible toasts as `clear-all`. Scoping `dismissAll` to a container is covered in [Multiple containers](/docs/vue/multiple-containers).

## Queue Controls

```ts [queue-controls.ts]
toast.pauseQueue();
toast.resumeQueue();
```

Affects all positions. Only controls queue processing — existing visible toasts are not affected.
